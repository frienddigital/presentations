Proposta Comercial

Integração VExpenses x SAP ECC para criação automática de requisições de compra

Proposta para a Airela, contemplando integração direta entre a plataforma VExpenses e o SAP ECC, com criação de requisições de compra a partir de despesas corporativas aprovadas.

Cliente: Airela
Proponente: Friend Consulting
Modelo: Projeto fechado
Prazo estimado: 4 a 5 semanas

1. Contexto

A Airela utiliza a plataforma VExpenses para controle de despesas corporativas, incluindo cartão corporativo e prestação de contas. Após a aprovação das despesas no VExpenses, existe a necessidade de automatizar a criação de requisições de compra no SAP ECC, reduzindo atividades manuais, retrabalho e riscos operacionais.

A Friend Consulting propõe o desenvolvimento de uma integração direta entre o VExpenses e o SAP ECC, considerando que a Airela não possui SAP PI/PO e que a API do VExpenses já está disponível para consumo.

2. Resumo executivo da Airela

A Airela é uma indústria farmacêutica brasileira, com operação em Pedras Grandes/SC, dedicada ao desenvolvimento, fabricação e comercialização de medicamentos e produtos voltados à saúde e bem-estar.

A empresa declara presença em mais de 80.000 pontos de venda, atendendo farmácias independentes, pequenas, médias e grandes redes, além de atuar com distribuidoras, canal hospitalar, marcas próprias e terceirização.

Portfólio e crescimento

3. Objetivo da proposta

Desenvolver uma integração que permita a criação automática de requisições de compra no SAP ECC a partir de despesas/prestações de contas aprovadas no VExpenses.

Resultado esperado: após a aprovação da despesa no VExpenses, o SAP ECC deverá criar automaticamente uma requisição de compra por colaborador/viagem, com cada despesa representada como um item da requisição.

4. Escopo macro da solução

Integração Consumo da API do VExpenses diretamente pelo SAP ECC, sem utilização de PI/PO, CPI ou middleware.
Requisição SAP Criação automática de requisição de compra via BAPI standard, com uma requisição por colaborador/viagem.
Itens da requisição Cada despesa aprovada no VExpenses será representada como um item da requisição no SAP.

Atividades contempladas

  • Alinhamento funcional e técnico do processo de integração.
  • Análise dos dados disponibilizados pela API do VExpenses.
  • Validação da conectividade entre SAP ECC e API VExpenses.
  • Definição dos campos necessários para criação da requisição no SAP.
  • Criação de tabela Z simples para tratamento de DE-PARA/exceções.
  • Desenvolvimento do consumo da API VExpenses pelo SAP ECC.
  • Tratamento dos dados por colaborador/viagem.
  • Criação dos itens da requisição a partir das despesas aprovadas.
  • Tratamento simples para despesas de combustível, conforme regra definida com a Airela.
  • Criação da requisição de compra via BAPI standard do SAP.
  • Criação de log simples de processamento.
  • Disponibilização de reprocessamento manual básico em caso de erro.
  • Testes unitários e apoio aos testes integrados com a Airela.
  • Apoio à homologação e go-live assistido.

8. Premissas

  • A Airela utiliza SAP ECC e não possui SAP PI/PO.
  • A integração será direta entre SAP ECC e API VExpenses.
  • A API do VExpenses já está disponível e documentada para uso no projeto.
  • A Airela será responsável pelas liberações de rede, certificados, firewall, permissões e acesso à API.
  • Todos os campos necessários para criação da requisição virão da API do VExpenses ou serão tratados por tabela Z simples.
  • Dados contábeis, grupo de compras e demais informações de imputação serão fornecidos pelo VExpenses/Airela.
  • O material padrão a ser utilizado na requisição já estará cadastrado no SAP MM.
  • O fluxo de aprovação seguirá a estratégia de liberação já existente no SAP.
  • O log será simples, com foco em status de processamento, mensagens de erro e reprocessamento manual.
  • A Airela disponibilizará usuários-chave para validação funcional, testes integrados e homologação.

9. Fora de escopo

  • Implantação de PI/PO, CPI, BTP Integration Suite ou qualquer middleware.
  • Configurações FI/CO.
  • Alterações na estratégia de liberação do SAP.
  • Criação automática de pedido de compra.
  • Lançamentos financeiros ou contábeis.
  • Retorno de status ou número da requisição para o VExpenses.
  • Anexação de comprovantes no SAP.
  • Alterações na plataforma VExpenses.
  • Alterações em cadastros mestres.
  • Cockpit avançado de monitoramento.
  • Suporte recorrente após go-live.
  • Mudanças futuras na API, layout, regras ou campos do VExpenses.
  • Regras adicionais não mapeadas na etapa de especificação funcional.

10. Critério de aceite

A entrega será considerada aceita quando uma despesa/prestação de contas aprovada no VExpenses gerar automaticamente no SAP ECC uma requisição de compra por colaborador/viagem, com seus respectivos itens, registro de log simples e possibilidade de reprocessamento manual em caso de erro.
